# ניהול משמרות - גן ילדים 🧸

מערכת לניהול משמרות ואילוצים בגן ילדים פרטי. עברית מלאה, RTL.

## הרצה עם Docker (מומלץ)

```bash
docker compose up --build
```

האפליקציה תהיה זמינה ב: **http://localhost:3100**

הנתונים נשמרים ב־volume בשם `scheduler-data` (קובץ SQLite). גיבוי = העתקת ה־volume / קובץ ה־DB.

## הרצה מקומית (פיתוח)

שני טרמינלים:

```bash
# טרמינל 1 - שרת
cd backend
npm install
npm run dev          # מאזין על :3100

# טרמינל 2 - ממשק
cd frontend
npm install
npm run dev          # http://localhost:5173 (פרוקסי ל-API)
```

## מבנה

- `backend/` — Node.js + Express + TypeScript + SQLite (better-sqlite3)
- `frontend/` — React + Vite + Tailwind (RTL)
- בייצור: ה־backend מגיש גם את ה־API (`/api/*`) וגם את ה־frontend הבנוי.

## מצב נוכחי

- ✅ ניהול צוות (קבועה / מנהלת, תבנית ימים+שעות, מכסות, הגדרות שישי)
- ✅ דרישות כיסוי הניתנות לעריכה (זרועות בערכי ברירת מחדל)
- ✅ שיבוץ שבועי: בחירת שבוע, הזנת אילוצים + ימי בלוז/חגים
- ✅ מנוע יצירת לו"ז: עיגון קבועות לפי תבנית, שיבוץ מכסת בקרים/צהריים של מנהלות לימים הנצרכים ביותר
- ✅ ניתוח כיסוי לכל פרוסת זמן (מאויש מול דרוש) עם הדגשת חוסרים
- ✅ הצעות החלפה: נשות צוות פנויות באותו יום, מדורגות לפי התאמה
- ✅ עריכה ידנית: הוספה/הסרה של משמרות ישירות בלוח (שורדות יצירה מחדש)
- ✅ אישור לו"ז (טיוטה → מאושר) עם נעילת עריכה, וביטול אישור
- ✅ הדפסה (פריסת נוף נקייה, ללא כלי עריכה)

## מנוע השיבוץ — לוגיקה

1. **עיגון קבועות**: כל אשת צוות קבועה משובצת לפי תבנית הימים+שעות שלה (בכפוף לאילוצים, ימי בלוז, וכללי שישי).
2. **שיבוץ מנהלות**: כל מנהלת משובצת את מלוא מכסת הבקרים/צהריים שלה (08:00–13:00 / 13:00–16:00), כל משמרת ליום עם החוסר הגדול ביותר באותו רגע. מנהלות משובצות בימים א'–ה' בלבד.
3. **ניתוח כיסוי**: לכל פרוסת זמן נספרות נשות הצוות הנוכחות מול הדרוש; פער = חוסר.
4. **החלפות**: לכל חוסר מוצעות נשות צוות שאינן עובדות באותו יום ואינן חסומות, מדורגות (עובדת בד"כ ביום ובשעות אלו → עובדת בד"כ ביום זה → פנויה).

### כללי שישי (לכל אשת צוות)
- **רגיל** — עובדת בשישי לפי התבנית.
- **לא רלוונטי** — לא עובדת בשישי.
- **בוקר בלבד** — זמינה למשמרת שישי (07:30–12:00).
- **פעם בשבועיים** — עובדת לסירוגין; הזוגיות מחושבת מתאריך עוגן (שישי ידוע שעבדה), עם אפשרות עקיפה ידנית.

## העלאה לאוויר (Railway)

הקוד מוכן לפריסה בענן. ה־Dockerfile מושך כברירת מחדל דימוי **ציבורי** (Docker Hub) ו־npm ציבורי,
כך שבענן הכל עובד; מקומית ה־`docker-compose` עוקף לרישום הפנימי (artifactory). העבודה המקומית לא נפגעת.

שלבים ב־Railway (אחרי חיבור GitHub):
1. **New Project → Deploy from GitHub repo** → בחר את ה־repo. Railway מזהה את ה־Dockerfile ובונה.
2. **Volume**: הוסף Volume וקבע Mount path = `/app/data` — כאן נשמר ה־SQLite (אחרת הנתונים יימחקו בכל פריסה).
3. **Variables**: הגדר (אופציונלי אך מומלץ) `ADMIN_USER`, `ADMIN_PASS`, `AUTH_TOKEN` — כדי שהסיסמה לא תהיה בקוד. אל תגדיר `PORT` (Railway מזריק אותו לבד).
4. Railway יספק כתובת `https://...` — זה האתר.
5. בכל `git push` ל־GitHub, Railway יבנה ויפרוס מחדש אוטומטית.
